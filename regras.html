1Visão Geral do Sistema
🎯 Objetivo

O dashboard cruza três fontes de dados para medir o resultado real do tráfego pago: quantas propostas geradas pelo marketing foram aprovadas, pagas ou reprovadas, quanto custou adquirir cada cliente e qual o retorno sobre o investimento.

✅
Processo diário: importe sempre o período completo que deseja analisar. O dashboard substitui os dados anteriores a cada importação — não há acúmulo de registros duplicados.
📂 As três fontes de dados
  • Facebook Ads (BM-03 e/ou BM-06): exportação de campanhas do Gerenciador de Anúncios. Fornece investimento e leads gerados.
  • Sistema Smart (.xlsx): exportação de atendimentos/leads. Fornece a origem (canal de marketing) e audiência de cada cliente via CPF/telefone.
  • Ecorban (.xlsx): exportação de propostas. É a fonte principal de status da proposta, valor e dados de vendedor/loja.

3Cruzamento de Dados: Ecorban ↔ Sistema Smart
🔗 Como o cruzamento funciona

Para cada proposta do Ecorban, o sistema tenta encontrar o mesmo cliente no Sistema Smart usando CPF (prioridade) e depois telefone. Quando encontrado, usa as informações de Origem e Audiência do Smart para classificar se a venda veio ou não do marketing.

🪪 Normalização de CPF
  • Remove tudo que não é número
  • Completa com zeros à esquerda até 11 dígitos — CPFs exportados pelo Excel como número perdem o zero inicial (ex: 01234567890 vira 1234567890 no Excel)
  • CPF 00000000000 é ignorado
📱 Normalização de Telefone
  • Remove tudo que não é número
  • Se houver múltiplos telefones separados por ;, usa apenas o primeiro
  • Remove o DDI 55 do início quando o número tiver mais de 11 dígitos
  • Números com menos de 8 dígitos são ignorados
📌 Regra: múltiplos registros no Smart para o mesmo CPF

Quando um CPF aparece mais de uma vez no Sistema Smart (cliente que entrou em contato em datas diferentes), o sistema usa sempre o registro mais antigo (menor "Data de Criação"). Isso garante que a origem original do cliente seja preservada.

ℹ️
Proposta não encontrada no Smart: vai para a fila de Revisão Manual. O gestor pode classificar manualmente se é ou não marketing. Essas classificações são exportadas em um arquivo .json e podem ser reimportadas no dia seguinte para não perder o trabalho.
4Classificação: É Marketing ou Não?
✅ É considerado MARKETING quando (qualquer uma dessas condições):
  • Campo Origem no Smart = Instagram
  • Campo Origem no Smart = Storie Instagram
  • Campo Origem no Smart = Sem Origem
  • Campo Origem = NÃO MAPEADO E campo Audiência = NÃO MAPEADO
❌ NÃO é marketing quando:
  • A Origem está preenchida com qualquer outro valor (ex: indicação, ligação, WhatsApp orgânico, etc.)
  • A Origem é NÃO MAPEADO mas a Audiência tem algum valor preenchido
⚠️
Revisão Manual: propostas não encontradas no Smart ficam com classificação em branco (null) e são excluídas dos KPIs de marketing até que o gestor as classifique manualmente na aba "Revisão Manual".

6Contagem de Leads — Facebook Ads
💬 Regra: somente linhas de "mensagem/conversa"

O relatório do Facebook Ads contém múltiplos tipos de resultado (cliques, alcance, conversas, etc.). O dashboard filtra apenas as linhas onde o campo "Tipo de resultado" contém as palavras mensagem ou conversa para contar leads — que são os atendimentos iniciados via anúncio.

Se o arquivo não tiver essa coluna preenchida, usa todas as linhas como fallback.

💰 Investimento

O investimento total é somado de todas as linhas do arquivo do Facebook, independente do tipo de resultado, usando a coluna Montante gasto (BRL).

Se houver dois arquivos (BM-03 + BM-06), os dados são somados automaticamente.

7Coluna de Valor Utilizada
💵 Qual valor é usado do Ecorban?

O dashboard usa a coluna Multiplicador do Ecorban como referência de valor das propostas — a mesma coluna destacada no relatório de Detalhamento do Ecorban. Essa coluna é buscada nesta ordem de prioridade:

  1. Coluna Multiplicador
  2. Coluna Valor Multiplicador
  3. Coluna Valor (fallback)

O painel de Diagnóstico (na tela de Importar) mostra os nomes exatos das colunas detectadas no arquivo — use para confirmar qual está sendo lida.

8Definição dos KPIs
CPL Calculado
Custo por Lead calculado internamente: Investimento ÷ Leads Gerados. Diferente do CPL reportado pelo Facebook (que é uma média das campanhas).
CAC
Custo de Aquisição de Cliente: Investimento ÷ Vendas Pagas de Marketing. Quanto custou converter um lead em venda efetivamente paga.
ROAS
Retorno sobre o Investimento em Anúncios: Valor das Vendas Pagas de Marketing ÷ Investimento. Quanto R$ de operações pagas cada R$ investido gerou.
Taxa de Conversão
Vendas Pagas de Marketing ÷ Leads Gerados × 100. Percentual de leads que se tornaram clientes pagos.
Ticket Médio
Valor Total das Vendas Pagas ÷ Quantidade de Vendas Pagas. Valor médio por operação. Calculado separadamente para Marketing e para Todas as Origens.
Em Andamento
Propostas com status Aprovado (processo ativo mas ainda não pago). Equivale às "Válidas em análise" do Ecorban.
Válidas (Total)
Em Andamento + Pagas. Equivale ao total da coluna "Válidas" no relatório de Detalhamento do Ecorban. Use esse número para conferir com o sistema.
9Esteira de Vendas — Como Ler
📊 A Esteira mostra dois blocos:
  • Tráfego Pago (Marketing): apenas propostas onde a origem foi identificada como marketing (Instagram, Storie Instagram, Sem Origem, ou ambos NÃO MAPEADO)
  • Todas as Origens: todas as propostas do Ecorban no período, independente da origem — inclui vendas orgânicas, indicações, etc.
🟡 Em Andamento

Propostas aprovadas que ainda estão em processo (análise, assinatura, integração bancária, etc.). Ainda podem virar Pagas ou Reprovadas.

🟢 Pagas

Operações efetivamente pagas/averbadas. São os status: Pago, Integrado, Desaverbação e Desaverbação Master.

🔴 Reprovadas

Propostas perdidas definitivamente por política interna, margem negativa, desistência do cliente, etc.

🔵 Válidas (Total)

Soma de Em Andamento + Pagas. Esse total deve bater com a coluna "Multiplicador" do relatório "Válidas" do Ecorban.

11Revisão Manual e Persistência
🔍 O que vai para revisão manual?
  • Propostas do Ecorban cujo CPF não foi encontrado no Sistema Smart
  • Propostas cujo telefone também não cruzou
  • Nesse caso, o sistema não consegue determinar se é marketing ou não
💾 Como persistir as classificações manuais entre sessões
  1. Na aba Revisão Manual, classifique as vendas como "É Marketing" ou "Não é"
  2. Clique em "Exportar Classificações (.json)" — salva um arquivo com todas as decisões por CPF
  3. No dia seguinte, na tela de Importar, carregue esse arquivo no campo "Classificações Manuais" antes de processar
  4. As classificações anteriores serão aplicadas automaticamente, sem precisar reclassificar
⚠️
A classificação manual tem prioridade absoluta sobre qualquer cruzamento automático. Se um CPF foi classificado manualmente, essa decisão prevalece mesmo que o Smart seja reimportado com dados diferentes.
12Metas e Indicadores de Desempenho
🎯 Como configurar as metas

Na aba Metas, configure os valores esperados para o período. As metas são salvas no navegador e persistem entre sessões (mesmo após fechar o arquivo). Não é necessário reconfigurar todos os dias.

13Painel de Diagnóstico
🔬 O que o diagnóstico mostra

Após processar, o painel de Diagnóstico (na tela de Importar) exibe:

  • Sistema Smart: quantos registros foram lidos, CPFs e telefones indexados, nomes das colunas detectadas
  • Ecorban: total de propostas, quantas cruzaram com o Smart (taxa de match), quantas foram para revisão, distribuição de status (Pago / Aprovado / Reprovado / Desconhecido)
  • Facebook: total de linhas por BM
ℹ️
Taxa de match abaixo de 50%: provável causa são CPFs com zeros à esquerda perdidos na exportação do Excel. Verifique se o Ecorban exporta o CPF como texto (com zeros) ou como número (sem zeros). O Sistema Smart precisa estar com o mesmo CPF.
